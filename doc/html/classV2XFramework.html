Detailed Description

Interface for a generic V2XFramework.

Each method represents a Callback which is called in specific moments of the simulation. These are inherited from ADynamicAPI, except for arrivalNewAP which is called from inside the init method. Right now, the only way to communicate with the GUI is through the finished GKModel when the simulation starts.

Right now only one way to communicate exists, while it is planned to have multiple way to carry on communications. This first and simple way is called "Simple", and is described in the class V2XSimpleAP.

Each object in this framework follows the pre and post rule: a pre method is called before any simulation step, while a post method is called after a simulation step has been done for all the objects.

Please remember to call its method init to correctly initialize the type names and folders for the framework.
